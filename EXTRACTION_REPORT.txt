1. NPC DATA STRUCTURE
=====================

Each NPC record contains 24 fields:

1. status           - "Alive" or "Dead"
2. role             - Knight, Page, Baron, Steward, Priest, etc.
3. name             - NPC's full name
4. glory            - Glory points (numeric, "-", "--", "See Sheet", etc.)
5. year_born        - Birth year
6. year_died        - Death year (null for living)
7. age              - Current age (null for some records)
8. pronoun          - Gender pronoun (e.g., "He/him", "She/her")
9. household        - Family affiliation (Blackwood, Cador, Dawnwell, Westwood, Upavon, or empty)
10. manor           - Location/Manor name
11. eligibility     - Marriage eligibility status
12. dowry           - Dowry information
13. notes           - Free-form notes about the NPC
14. passions        - Passions and their levels (multi-line text)
15. skills          - Skills and their levels (multi-line text)
16. stats           - Character stats
17. blessed         - Boolean, whether blessed
18. blessed_note    - Notes on blessing
19. page_placed     - Boolean, whether placed as a page
20. page_court      - Where placed as page
21. training_path   - Training path information
22. training_where  - Where trained
23. came_of_age     - Boolean, whether came of age
24. retired         - Boolean, whether retired

6. DATA STORAGE
===============

The application uses localStorage with these keys:

- bl_living      : Current living NPCs (JSON array)
- bl_dead        : Current dead NPCs (JSON array)
- bl_households  : Household definitions (JSON array)
- bl_factions    : Faction definitions (JSON array)
- bl_year        : Current game year

Note: On startup, the application loads from localStorage if available,
otherwise falls back to SEED_* constants embedded in the HTML.


7. EXPORT MECHANISM
===================

The exportLedger() function:
1. Serializes current LIVING, DEAD, HOUSEHOLDS, and FACTIONS arrays
2. Extracts the full page HTML from document.documentElement.outerHTML
3. Replaces the seed data placeholders with current data
4. Saves as HTML file with filename: britannia_ledger_v1.7_YYYYMMDD.html
5. This maintains backward compatibility - exported file is identical to original
